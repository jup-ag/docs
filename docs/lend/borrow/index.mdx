Jupiter Borrow is the collateralized borrowing side of Jupiter Lend. Users create positions by depositing collateral into vaults, then borrow against that collateral. Each vault has a supply token (collateral) and a borrow token. Positions are represented as NFTs, and the protocol tracks collateral, debt, tick-based pricing, and liquidation status. Users can deposit, borrow, repay, withdraw, and liquidators can liquidate undercollateralized positions. Integration is done via the Jupiter Lend SDK for transactions and the Read SDK for vault and position data.

## Key data structures

<CodeGroup>

```typescript Typescript
// Vault config (per vault)
type VaultConfig = {
  vaultId: number;
  supplyToken: PublicKey;   // Collateral mint
  borrowToken: PublicKey;   // Borrow asset mint
  oracle: PublicKey;
  collateralFactor: number;
  liquidationThreshold: number;
  liquidationPenalty: number;
  borrowFee: number;
  // ...
};

// User position (raw on-chain)
type UserPosition = {
  vaultId: number;
  nftId: number;
  positionMint: PublicKey;
  tick: number;
  tickId: number;
  supplyAmount: BN;
  dustDebtAmount: BN;
  isSupplyOnlyPosition: number | boolean;
  // ...
};

// Decoded position (with exchange prices)
type NftPosition = {
  nftId: number;
  owner: PublicKey;
  supply: BN;
  borrow: BN;
  dustBorrow: BN;
  tick: number;
  tickId: number;
  isLiquidated: boolean;
  // ...
};
```

</CodeGroup>

## Vault and position model

- **Vault** — A market with a supply token (collateral) and borrow token. Each vault has config (oracle, risk params), state (totals, exchange prices), and limits (withdraw, borrow availability).
- **Position** — A user's collateralized borrow. Identified by vault ID and position (NFT) id. Tracks collateral (supply), debt (borrow), tick, and liquidation status.
- **Tick** — Price ticks determine the borrow amount for a given collateral amount. Positions use tick-based pricing for leverage.
