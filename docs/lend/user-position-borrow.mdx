This page covers **Borrow** position data: vault positions (collateral, debt, tick, liquidation status). You need a `vaultId` and `positionId` (the NFT id from [Create position](/docs/lend/create-position)). For **Earn** position data (supply/shares per asset), see [User position (Earn)](/docs/lend/user-position).

<Note>
  All position data here is derived from on-chain accounts using the Borrow SDK (`@jup-ag/lend/borrow`). Amounts are returned in **raw** form (internal precision); scale by the vault's exchange prices (e.g. 1e12) for human-readable collateral and debt.
</Note>

---

## Reading Borrow position state

<Steps>
  <Step title="Get current position state">
    Fetch the current state of a vault position: collateral, debt, tick, and liquidation status.

    ```typescript
    import { Connection } from "@solana/web3.js";
    import { getCurrentPosition } from "@jup-ag/lend/borrow";

    const connection = new Connection("https://api.mainnet-beta.solana.com");

    const position = await getCurrentPosition({
      vaultId: 1,
      positionId: 0,   // nftId from Create position
      connection,
    });

    console.log({
      tick: position.tick,
      tickId: position.tickId,
      colRaw: position.colRaw.toString(),
      debtRaw: position.debtRaw.toString(),
      dustDebtRaw: position.dustDebtRaw.toString(),
      finalAmount: position.finalAmount.toString(),
      isSupplyOnlyPosition: position.isSupplyOnlyPosition,
      userLiquidationStatus: position.userLiquidationStatus,
      postLiquidationBranchId: position.postLiquidationBranchId,
    });
    ```

    <Info>
      - **colRaw** / **debtRaw** – Collateral and debt in raw (scaled) form. Divide by the vault's supply/borrow exchange price (e.g. 1e12) and token decimals for display.
      - **isSupplyOnlyPosition** – `true` if the position has no debt (supply-only or fully liquidated).
      - **userLiquidationStatus** – `true` if the position has been (partially or fully) liquidated.
    </Info>
  </Step>
  <Step title="Preview position after an operate">
    Use `getFinalPosition` to compute the position state **after** a hypothetical deposit, withdraw, borrow, or repay (without sending a transaction).

    ```typescript
    import { Connection, PublicKey } from "@solana/web3.js";
    import BN from "bn.js";
    import {
      getCurrentPosition,
      getFinalPosition,
    } from "@jup-ag/lend/borrow";

    const connection = new Connection("https://api.mainnet-beta.solana.com");
    const signer = new PublicKey("YourWalletPublicKey");

    const currentPosition = await getCurrentPosition({
      vaultId: 1,
      positionId: 0,
      connection,
    });

    // Example: preview after borrowing 100 units (in token base units, e.g. 100_000 for 0.1 USDC)
    const borrowAmount = new BN(100_000);
    const finalPosition = await getFinalPosition({
      vaultId: 1,
      currentPosition,
      newColAmount: new BN(0),
      newDebtAmount: borrowAmount,
      connection,
      signer,
    });

    console.log("After borrow:", {
      tick: finalPosition.tick,
      colRaw: finalPosition.colRaw.toString(),
      debtRaw: finalPosition.debtRaw.toString(),
      isSupplyOnlyPosition: finalPosition.isSupplyOnlyPosition,
    });
    ```

    <Note>
      For deposit use `newColAmount` &gt; 0 and `newDebtAmount` = 0. For withdraw use negative `newColAmount`. For repay use negative `newDebtAmount` (e.g. `repayAmount.neg()`).
    </Note>
  </Step>
</Steps>

## Borrow position state structure

When calling `getCurrentPosition`, the returned object contains:

| Field                    | Type    | Description                                                                 |
| ------------------------ | ------- | --------------------------------------------------------------------------- |
| `tick`                   | number  | Risk tick of the position (ratio of debt to collateral).                   |
| `tickId`                 | number  | Tick id for this position (used when the tick has been liquidated).         |
| `colRaw`                 | BN      | Collateral in raw (scaled) form. Scale by vault supply exchange price.      |
| `debtRaw`                | BN      | Debt in raw (scaled) form. Scale by vault borrow exchange price.            |
| `dustDebtRaw`            | BN      | Dust debt amount (internal).                                                |
| `finalAmount`            | BN      | Effective collateral (or 0 if no net debt).                                 |
| `isSupplyOnlyPosition`   | boolean | `true` if position has no debt (supply-only or fully liquidated).            |
| `userLiquidationStatus`  | boolean | `true` if the position has been liquidated (partial or full).                |
| `postLiquidationBranchId`| number  | Branch id after liquidation (for internal/liquidation tracking).             |

### Scaling raw amounts for display

Raw amounts use the protocol's internal precision (e.g. 1e12). To show collateral and debt in token units, divide by the vault's supply/borrow exchange price and adjust for token decimals. Exchange prices are available from the vault's view instruction or from your app's API; the Borrow SDK uses them internally in `getFinalPosition` and in instruction building.

## Optional: program instance for advanced use

If you need the Anchor `Program` instance (e.g. to fetch vault config or state directly), use `getVaultsProgram`:

```typescript
import { Connection, PublicKey } from "@solana/web3.js";
import { getVaultsProgram } from "@jup-ag/lend/borrow";
import { borrowPda } from "@jup-ag/lend";

const program = getVaultsProgram({
  connection: new Connection("https://api.mainnet-beta.solana.com"),
  signer: new PublicKey("YourWalletPublicKey"),
});

const vaultState = await program.account.vaultState.fetch(
  borrowPda.getVaultState(vaultId),
);
```
