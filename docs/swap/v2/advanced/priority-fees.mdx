Control how much you pay for transaction priority on `/order`. Without these parameters, Jupiter automatically sets an appropriate priority fee based on current network conditions.

## Parameters

| Parameter | Description |
| --- | --- |
| `priorityFeeLamports` | Priority fee in lamports. Overrides automatic optimisation. |
| `jitoTipLamports` | Jito MEV tip in lamports for faster block inclusion. |
| `broadcastFeeType` | `maxCap` (fee is a maximum) or `exactFee` (exact amount). Ignored if no priority/tip fees set. |

## When to set manually

- **High-value or time-sensitive swaps** where faster inclusion is worth the cost
- **Trading bots** that need predictable fee behaviour
- **During network congestion** when automatic fees may not be aggressive enough

For most integrations, leave these unset and let Jupiter optimise automatically.
