The Swap API V2 routes through multiple liquidity sources. Which parameters you pass to `/order` determines which routers are eligible, directly affecting the price you get.

## Routing overview

By default, `/order` with no optional parameters enables all routers:

| Router | Type | Description |
| --- | --- | --- |
| **Metis (Iris)** | Onchain aggregator | Jupiter's core routing engine |
| **JupiterZ** | RFQ (Request for Quote) | Market makers providing off-chain liquidity. Often beats onchain by 5-20bps on major pairs. |
| **Dflow** | Third-party | - |
| **OKX** | Third-party | - |

The `/build` endpoint always uses **Metis only**. RFQ and third-party routers are not available.

The response `router` field tells you which router won: `iris` (Metis), `jupiterz`, `dflow`, or `okx`.

## Routing impact matrix

This table shows which routers are available for each parameter combination on `/order`.

| Parameter | Metis | JupiterZ (RFQ) | Dflow | OKX |
| --- | --- | --- | --- |--- |
| **(no optional params)** | Yes | Yes | Yes | Yes |
| `receiver` | Yes | **No** | Yes | Yes |
| `referralAccount` | Yes | **No** | Yes | Yes |
| `payer` (integrator gasless) | Yes | **No** | **No** | **No** |
| `excludeRouters: jupiterz` | Yes | **No** | Yes | Yes |

## `/order` parameters

### Required

| Parameter | Type | Description |
| --- | --- | --- |
| `inputMint` | string | Mint address of the input token |
| `outputMint` | string | Mint address of the output token |
| `amount` | string | Amount in smallest unit of the input token |

### Taker and recipient

| Parameter | Type | Default | Routing | Description |
| --- | --- | --- | --- | --- |
| `taker` | string | - | All routers | Wallet address that will sign the transaction. Required to receive an assembled transaction. Without it, you get a quote only. |
| `receiver` | string | - | Metis only | Override output token recipient. Must differ from taker. Expects a wallet address (not a token account). For non-SOL output, tokens go to the receiver's ATA. Disables RFQ routing. |

### Swap behaviour

| Parameter | Type | Default | Routing | Description |
| --- | --- | --- | --- | --- |
| `swapMode` | ExactIn | ExactIn | All routers | Swap mode. Currently only `ExactIn` is supported. |
| `slippageBps` | integer (0-10000) | auto | All routers | Slippage tolerance in basis points. If not set, Jupiter automatically determines an appropriate value. |
| `excludeRouters` | string | - | Controls routing | Comma-separated routers to exclude. Available: `iris` (Metis), `jupiterz` (RFQ), `dflow`, `okx`. |
| `excludeDexes` | string | - | Iris only | Comma-separated DEXes to exclude from the Iris (Metis) router. Does not affect other routers. |
### Integrator fees

<Warning>
  Adding fee parameters disables RFQ routing (JupiterZ). You will only get Metis quotes, which may result in worse pricing on major pairs.
</Warning>

| Parameter | Type | Default | Routing | Description |
| --- | --- | --- | --- | --- |
| `referralAccount` | string | - | Metis only | Referral account address for the Jupiter referral project. Must be used with `referralFee`. See [Referral Program](/tool-kits/referral-program). |
| `referralFee` | number (50-255) | - | Metis only | Referral fee in basis points. Must be used with `referralAccount`. |

### Gas and priority

| Parameter | Type | Default | Routing | Description |
| --- | --- | --- | --- | --- |
| `payer` | string | - | Iris only | Account that covers all gas-related fees (signature, priority, rent) on behalf of the taker. Routes through Iris only. Transaction requires both taker and payer signatures. See [Gasless Swaps](/docs/swap/v2/advanced/gasless) for details. |
| `priorityFeeLamports` | number | auto | All routers | Priority fee in lamports. If not set, Jupiter automatically optimises. |
| `jitoTipLamports` | number | - | All routers | Jito MEV tip in lamports for faster block inclusion. |
| `broadcastFeeType` | maxCap, exactFee | - | All routers | `maxCap` treats fee as a maximum, `exactFee` uses exact amount. Ignored if no priority/tip fees set. |

## `/build` parameters

All `/build` requests use Metis-only routing. No RFQ or third-party routers.

### Required

| Parameter | Type | Description |
| --- | --- | --- |
| `inputMint` | string | Mint address of the input token |
| `outputMint` | string | Mint address of the output token |
| `amount` | string | Amount in smallest unit of the input token |
| `userPublicKey` | string | Your wallet address |

### Optional

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `slippageBps` | integer (0-10000) | 50 | Slippage tolerance in basis points |
| `mode` | "fast" | (default) | Reduced latency routing (trades optimality for speed) |
| `maxAccounts` | integer (1-64) | 64 | Maximum accounts for the swap route |
| `platformFeeBps` | integer (0-10000) | 0 | Integrator platform fee. Requires `feeAccount`. |
| `feeAccount` | string | - | Token account to collect platform fees |
| `payer` | string | userPublicKey | Account paying transaction fees and rent |
| `wrapAndUnwrapSol` | boolean | true | Auto wrap/unwrap SOL |
| `dexes` | string | (all) | Restrict to specific DEXes. Mutually exclusive with `excludeDexes`. |
| `excludeDexes` | string | (none) | Exclude specific DEXes. Mutually exclusive with `dexes`. |
| `destinationTokenAccount` | string | - | SPL token account for output |
| `nativeDestinationAccount` | string | - | Native SOL account for output |
| `blockhashSlotsToExpiry` | integer (1-300) | 150 | Slots until blockhash expires |

## Mode: ultra vs manual

The `/order` response includes a `mode` field:

- **"ultra"**: No optional params used. All routers competed. Best price.
- **"manual"**: Optional params detected. Routing may be restricted.

This field is informational. It helps you verify that your request got the routing behaviour you expected.
