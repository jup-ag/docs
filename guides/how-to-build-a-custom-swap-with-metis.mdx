## TL;DR

Metis is Jupiter's routing engine, powering every swap on [jup.ag](https://jup.ag). Get a quote, build a transaction, sign and send. Three API calls for a complete swap with full control over routing, fees, and execution.

```ts
// 1. Get quote
const quote = await fetch('https://api.jup.ag/swap/v1/quote?inputMint=So11...&outputMint=EPjF...&amount=100000000&slippageBps=50');

// 2. Build transaction
const swap = await fetch('https://api.jup.ag/swap/v1/swap', {
  method: 'POST',
  body: JSON.stringify({ quoteResponse: await quote.json(), userPublicKey: wallet.publicKey.toString() }),
});

// 3. Sign and send
const { swapTransaction } = await swap.json();
const tx = VersionedTransaction.deserialize(Buffer.from(swapTransaction, 'base64'));
tx.sign([wallet]);
await connection.sendRawTransaction(tx.serialize());
```

## When to use Metis

[Ultra](/docs/ultra/index) handles everything for you: routing, slippage, MEV protection, and transaction sending. No RPC node required.

Metis gives you full control. Use Metis when you need:

- **Custom instructions** in the same transaction (token transfers, program calls, memos)
- **CPI from your Solana program** to call Jupiter's swap on-chain
- **Your own transaction broadcasting** via your RPC or Jito
- **Full control over fees** including priority fees, compute budget, and platform fees
- **Route filtering** to include or exclude specific DEXes

Metis and Ultra are not mutually exclusive. You can use Ultra for simple swaps and Metis for flows that need composability.

## Core flow

### Step 1: Get a quote

`GET https://api.jup.ag/swap/v1/quote` returns the best route for a token pair. The quote response is passed directly to `/swap` or `/swap-instructions` in the next step.

```ts
const quoteResponse = await (
  await fetch(
    'https://api.jup.ag/swap/v1/quote?' + new URLSearchParams({
      inputMint: 'So11111111111111111111111111111111111111112',
      outputMint: 'EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v',
      amount: '100000000', // 1 SOL in lamports
      slippageBps: '50',   // 0.5%
    }),
    {
      headers: { 'x-api-key': 'YOUR_API_KEY' },
    }
  )
).json();
```

**Required parameters:**

| Parameter | Description |
|-----------|-------------|
| `inputMint` | Token mint address to swap from |
| `outputMint` | Token mint address to swap to |
| `amount` | Raw amount in smallest units (lamports for SOL, atomic units for other tokens) |
| `slippageBps` | Maximum slippage tolerance in basis points (50 = 0.5%) |

**Key optional parameters:**

| Parameter | Description |
|-----------|-------------|
| `platformFeeBps` | Basis points to charge as a platform fee. Requires `feeAccount` in `/swap` |
| `maxAccounts` | Limit total accounts in the swap instruction. Useful when composing transactions with additional instructions. Default: 64 |
| `dexes` | Comma-separated list of DEXes to route through exclusively. Use `/program-id-to-label` to get DEX names |
| `excludeDexes` | Comma-separated list of DEXes to exclude from routing |

<Note>
Always pass `instructionVersion=V2` unless you have a specific reason not to. V2 instructions support features like `nativeDestinationAccount` for receiving native SOL.
</Note>

**Understanding the response:**

```json expandable

  "outputMint": "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",
  "outAmount": "8730544",
  "otherAmountThreshold": "8686892",
  "swapMode": "ExactIn",
  "slippageBps": 50,
  "platformFee": null,

  "routePlan": [
    {
      "swapInfo": {
        "ammKey": "8sKQHfjNhvmAw94PhfvfMcytmqW6jmxvwieYyzXCCPu",
        "label": "HumidiFi",
        "inputMint": "So11111111111111111111111111111111111111112",
        "outputMint": "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",
        "inAmount": "100000000",
        "outAmount": "8730544"
```

Key fields:
- `outAmount`: Best output amount from routing (before slippage)
- `otherAmountThreshold`: Minimum output after applying `slippageBps`
- `routePlan`: The route breakdown showing which DEXes are used and the percentage split
- `priceImpactPct`: Price impact as a percentage string
- `mostReliableAmmsQuoteReport`: Quotes from the most reliable AMMs for the same pair, useful for debugging route decisions (see [Optimise execution](#route-debugging-with-mostreliableammsquotereport))

### Step 2: Build the transaction

`POST https://api.jup.ag/swap/v1/swap` takes the quote response and returns a serialised transaction ready for signing.

```ts
const swapResponse = await (
  await fetch('https://api.jup.ag/swap/v1/swap', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'YOUR_API_KEY',
    },
    body: JSON.stringify({
      quoteResponse,
      userPublicKey: wallet.publicKey.toString(),
      wrapAndUnwrapSol: true,
      prioritizationFeeLamports: {
        priorityLevelWithMaxLamports: {
          priorityLevel: 'veryHigh',
          maxLamports: 1000000,
        },
      },
    }),
  })
).json();
```

**Key `/swap` parameters:**

| Parameter | Default | Description |
|-----------|---------|-------------|
| `userPublicKey` | required | Wallet public key that will sign the transaction |
| `wrapAndUnwrapSol` | `true` | Automatically wrap SOL to wSOL before swapping and unwrap after. Set to `false` if you manage wSOL yourself |
| `prioritizationFeeLamports` | auto | Priority fee configuration. Use `priorityLevelWithMaxLamports` for estimated fees with a cap, or `jitoTipLamports` for Jito bundle tips |
| `feeAccount` | none | Token account to receive platform fees. Required if `platformFeeBps` was set in `/quote` |

#### Priority fees

Jupiter estimates priority fees using the local fee market across recent slots. You set a `priorityLevel` and a `maxLamports` cap to prevent overpaying:

```ts
prioritizationFeeLamports: {
  priorityLevelWithMaxLamports: {
    priorityLevel: 'veryHigh', // medium (25th %), high (50th %), veryHigh (75th %)
    maxLamports: 1000000,      // Cap in lamports
  },
}
```

For Jito bundle tips instead of priority fees:

```ts
prioritizationFeeLamports: {
  jitoTipLamports: 1000000, // Fixed tip amount (not a cap)
}
```

<Note>
You cannot use both `priorityLevelWithMaxLamports` and `jitoTipLamports` in the same `/swap` call. If you need both priority fees and Jito tips, use `/swap-instructions` to compose the transaction yourself.
</Note>

### Step 3: Sign and send

The `/swap` response contains a base64-encoded serialised transaction. Deserialise it, sign with your wallet, and send to the network.

```ts
import {
  Connection,
  Keypair,
  VersionedTransaction,
} from '@solana/web3.js';

const connection = new Connection('YOUR_RPC_ENDPOINT');

// Deserialize the transaction
const transactionBase64 = swapResponse.swapTransaction;
const transaction = VersionedTransaction.deserialize(
  Buffer.from(transactionBase64, 'base64')
);

// Sign
transaction.sign([wallet]);

// Send
const signature = await connection.sendRawTransaction(transaction.serialize(), {
  maxRetries: 2,
  skipPreflight: true,
});

console.log(`Transaction sent: https://solscan.io/tx/${signature}`);

// Confirm
const confirmation = await connection.confirmTransaction(
  { signature },
  'finalized'
);

if (confirmation.value.err) {
  throw new Error(`Transaction failed: ${JSON.stringify(confirmation.value.err)}`);
}
```

<Tip>
Set `skipPreflight: true` to avoid the extra RPC round-trip for preflight simulation. The transaction has already been simulated by Jupiter's backend when building the swap.
</Tip>

### Step 4: Handle errors

Common swap errors and how to handle them:

| Error | Cause | Fix |
|-------|-------|-----|
| `INSUFFICIENT_FUNDS` | Wallet does not have enough input tokens | Check balance before quoting |
| `SLIPPAGE_TOLERANCE_EXCEEDED` | Price moved beyond `slippageBps` | Re-quote and retry with fresh pricing |
| `Transaction expired` | Blockhash expired before confirmation | Re-quote immediately and send faster |
| `No route found` | No liquidity path between the token pair | Check if the token is tradeable on Jupiter |

**Retry pattern:**

```ts
async function swapWithRetry(maxRetries = 3) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      // Always re-quote on each attempt for fresh pricing
      const quoteResponse = await getQuote();
      const swapResponse = await buildSwap(quoteResponse);
      return await signAndSend(swapResponse);
    } catch (error) {
      if (attempt === maxRetries) throw error;
      console.log(`Attempt ${attempt} failed, retrying...`);
    }
  }
}
```

<Warning>
Always re-quote before retrying. Reusing a stale quote can lead to slippage failures or worse execution prices.
</Warning>

## Optimise execution

### Transaction size management with maxAccounts

When composing transactions with additional instructions (custom logic, memos, token transfers), you may exceed the transaction size limit. Use `maxAccounts` in `/quote` to constrain the swap instruction:

```ts
const quoteResponse = await fetch(
  'https://api.jup.ag/swap/v1/quote?' + new URLSearchParams({
    inputMint: 'So11111111111111111111111111111111111111112',
    outputMint: 'EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v',
    amount: '100000000',
    slippageBps: '50',
    maxAccounts: '40', // Leave room for your custom instructions
  }),
  { headers: { 'x-api-key': 'YOUR_API_KEY' } }
);
```

<Warning>
Setting `maxAccounts` too low reduces the available DEXes and can result in worse routes. Keep the value as high as possible while fitting your transaction. A value of 64 is a good starting point; only reduce if you hit size limits.
</Warning>

### Route debugging with mostReliableAmmsQuoteReport

The quote response includes `mostReliableAmmsQuoteReport`, which shows what the most reliable AMMs would have quoted for the same pair. This is useful for understanding route decisions:

```json
"mostReliableAmmsQuoteReport": {
  "info": {
    "BZtgQEyS6eXUXicYPHecYQ7PybqodXQMvkjUbP4R8mUU": "8729031",
    "Czfq3xZZDmsdGdUyrNLtRhGc47cXcZtLG4crryfu44zE": "8724328"
  }
}
```

Each key is an AMM account address and the value is the quoted output amount. Compare these with the chosen route's `outAmount` to verify the router selected the best path. If the chosen route returns less than a reliable AMM quote, it may indicate the route involves a multi-hop path with higher fees that offset the apparent advantage.

You can map AMM keys to DEX labels using `GET https://api.jup.ag/swap/v1/program-id-to-label`.

## Advanced use cases

### Add custom instructions with /swap-instructions

Use `POST https://api.jup.ag/swap/v1/swap-instructions` instead of `/swap` when you need to compose the swap with your own instructions. The endpoint returns individual instructions rather than a serialised transaction:

```ts
const instructionsResponse = await (
  await fetch('https://api.jup.ag/swap/v1/swap-instructions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'YOUR_API_KEY',
    },
    body: JSON.stringify({
      quoteResponse,
      userPublicKey: wallet.publicKey.toString(),
    }),
  })
).json();

if (instructionsResponse.error) {
  throw new Error('Failed to get swap instructions: ' + instructionsResponse.error);
}

const {
  computeBudgetInstructions,
  setupInstructions,
  swapInstruction,
  cleanupInstruction,
  addressLookupTableAddresses,
} = instructionsResponse;
```

**Response fields:**

| Field | Description |
|-------|-------------|
| `computeBudgetInstructions` | Compute budget and priority fee instructions |
| `setupInstructions` | Create any missing token accounts for the user |
| `swapInstruction` | The actual Jupiter swap instruction |
| `cleanupInstruction` | Unwrap wSOL back to native SOL (when `wrapAndUnwrapSol` is true) |
| `addressLookupTableAddresses` | Lookup table addresses for versioned transactions |

Build the transaction with your custom instructions mixed in:

```ts
import {
  Connection,
  PublicKey,
  TransactionInstruction,
  TransactionMessage,
  VersionedTransaction,
  AddressLookupTableAccount,
} from '@solana/web3.js';

const connection = new Connection('YOUR_RPC_ENDPOINT');

// Helper: convert API instruction to TransactionInstruction
function deserializeInstruction(instruction: any): TransactionInstruction {
  return new TransactionInstruction({
    programId: new PublicKey(instruction.programId),
    keys: instruction.accounts.map((key: any) => ({
      pubkey: new PublicKey(key.pubkey),
      isSigner: key.isSigner,
      isWritable: key.isWritable,
    })),
    data: Buffer.from(instruction.data, 'base64'),
  });
}

// Helper: resolve address lookup tables
async function getAddressLookupTableAccounts(
  keys: string[]
): Promise<AddressLookupTableAccount[]> {
  const infos = await connection.getMultipleAccountsInfo(
    keys.map((key) => new PublicKey(key))
  );
  return infos.reduce((acc, info, i) => {
    if (info) {
      acc.push(new AddressLookupTableAccount({
        key: new PublicKey(keys[i]),
        state: AddressLookupTableAccount.deserialize(info.data),
      }));
    }
    return acc;
  }, [] as AddressLookupTableAccount[]);
}

// Resolve lookup tables
const lookupTableAccounts = await getAddressLookupTableAccounts(
  addressLookupTableAddresses
);

// Build the transaction with custom instructions
const { blockhash } = await connection.getLatestBlockhash();

const messageV0 = new TransactionMessage({
  payerKey: wallet.publicKey,
  recentBlockhash: blockhash,
  instructions: [
    // Compute budget (from Jupiter)
    ...computeBudgetInstructions.map(deserializeInstruction),

    // Setup token accounts (from Jupiter)
    ...setupInstructions.map(deserializeInstruction),

    // YOUR CUSTOM INSTRUCTIONS GO HERE
    // e.g. a memo, token transfer, or program call

    // The actual swap
    deserializeInstruction(swapInstruction),

    // Cleanup / unwrap SOL (from Jupiter)
    ...(cleanupInstruction ? [deserializeInstruction(cleanupInstruction)] : []),
  ],
}).compileToV0Message(lookupTableAccounts);

const transaction = new VersionedTransaction(messageV0);
transaction.sign([wallet]);
await connection.sendRawTransaction(transaction.serialize(), { skipPreflight: true });
```

<Note>
When adding custom instructions, use `maxAccounts` in `/quote` to leave room in the transaction. If your transaction exceeds the size limit, lower `maxAccounts` and re-quote.
</Note>

### CPI from your Solana program

You can call Jupiter's swap from your own on-chain program via Cross-Program Invocation (CPI). The flow: get a quote client-side, fetch `/swap-instructions`, extract the `swapInstruction`, and wrap it in a CPI call.

#### Client side

```ts
// 1. Get quote
const quoteResponse = await (
  await fetch(
    'https://api.jup.ag/swap/v1/quote?' + new URLSearchParams({
      inputMint: 'So11111111111111111111111111111111111111112',
      outputMint: 'EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v',
      amount: '100000000',
      slippageBps: '50',
    }),
    { headers: { 'x-api-key': 'YOUR_API_KEY' } }
  )
).json();

// 2. Get swap instructions
const swapInstructionsResponse = await (
  await fetch('https://api.jup.ag/swap/v1/swap-instructions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'YOUR_API_KEY',
    },
    body: JSON.stringify({
      quoteResponse,
      userPublicKey: wallet.publicKey.toString(),
    }),
  })
).json();

// 3. Build your CPI instruction wrapping Jupiter's swap
const jupiterSwapIx = swapInstructionsResponse.swapInstruction;
const jupiterProgramId = new PublicKey(jupiterSwapIx.programId);
const jupiterData = Buffer.from(jupiterSwapIx.data, 'base64');
const jupiterAccounts = jupiterSwapIx.accounts.map((acc: any) => ({
  pubkey: new PublicKey(acc.pubkey),
  isSigner: acc.isSigner,
  isWritable: acc.isWritable,
}));

// Your instruction data: [discriminator] [jupiter_program_id: 32 bytes] [jupiter_data]
const cpiInstructionData = Buffer.concat([
  Buffer.from([2]), // Your program's discriminator for "swap via Jupiter"
  jupiterProgramId.toBytes(),
  jupiterData,
]);

const cpiInstruction = new TransactionInstruction({
  programId: YOUR_PROGRAM_ID,
  keys: jupiterAccounts,
  data: cpiInstructionData,
});

// 4. Build transaction
const lookupTableAccounts = await getAddressLookupTableAccounts(
  swapInstructionsResponse.addressLookupTableAddresses
);

const { blockhash } = await connection.getLatestBlockhash();
const messageV0 = new TransactionMessage({
  payerKey: wallet.publicKey,
  recentBlockhash: blockhash,
  instructions: [
    // Compute budget (set your own based on your program's needs)
    // Setup instructions (optional, include as separate instructions if needed)
    cpiInstruction,
    // Cleanup instruction (optional, include as separate instruction if needed)
  ],
}).compileToV0Message(lookupTableAccounts);

const transaction = new VersionedTransaction(messageV0);
transaction.sign([wallet]);
await connection.sendRawTransaction(transaction.serialize(), { skipPreflight: true });
```

#### On-chain program (Rust)

Your program receives the instruction data, extracts the Jupiter program ID and instruction data, then calls `invoke()`:

```rust
use solana_program::{
    account_info::AccountInfo,
    entrypoint::ProgramResult,
    instruction::{AccountMeta, Instruction},
    program::invoke,
    pubkey::Pubkey,
};

pub fn jupiter_swap_cpi(
    _program_id: &Pubkey,
    accounts: &[AccountInfo],
    instruction_data: &[u8],
) -> ProgramResult {
    // Parse: [jupiter_program_id: 32 bytes] [jupiter_instruction_data: remaining]
    let jupiter_program_id = Pubkey::new_from_array(
        instruction_data[..32].try_into().unwrap()
    );
    let jupiter_data = &instruction_data[32..];

    // Build account metas from the provided accounts
    let account_metas: Vec<AccountMeta> = accounts
        .iter()
        .map(|acc| {
            if acc.is_writable {
                AccountMeta::new(*acc.key, acc.is_signer)
            } else {
                AccountMeta::new_readonly(*acc.key, acc.is_signer)
            }
        })
        .collect();

    // CPI into Jupiter
    let ix = Instruction {
        program_id: jupiter_program_id,
        accounts: account_metas,
        data: jupiter_data.to_vec(),
    };

    invoke(&ix, accounts)
}
```

#### CPI considerations

- **Only use `swapInstruction`** from `/swap-instructions`. The `computeBudgetInstructions` are sized for a standalone swap, not your program. Set your own compute budget based on your full transaction.
- **`setupInstructions` and `cleanupInstruction`** handle token account creation and SOL unwrapping. Include these as separate instructions in the outer transaction if needed, not inside the CPI.
- **Priority fees** should be calculated based on your full transaction, not just the swap.
- **CPI account limit:** Solana currently supports up to **128 accounts** in CPI calls. A future upgrade ([SIMD-0339](https://github.com/solana-foundation/solana-improvement-documents/blob/main/proposals/0339-increase-the-account-limit-per-transaction.md)) will raise the limit to 255.

<Tip>
For complete CPI examples, see [jup-ag/jupiter-cpi-swap-example](https://github.com/jup-ag/jupiter-cpi-swap-example) and [jup-ag/sol-swap-cpi](https://github.com/jup-ag/sol-swap-cpi). For the `jupiter-cpi` Rust crate, see [jup-ag/jupiter-cpi](https://github.com/jup-ag/jupiter-cpi).
</Tip>

### Collect platform fees

Charge a fee on swaps through your integration. Pass `platformFeeBps` in the quote and `feeAccount` in the swap:

```ts
// 1. Quote with platform fee
const quoteResponse = await (
  await fetch(
    'https://api.jup.ag/swap/v1/quote?' + new URLSearchParams({
      inputMint: 'So11111111111111111111111111111111111111112',
      outputMint: 'EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v',
      amount: '100000000',
      slippageBps: '50',
      platformFeeBps: '50', // 0.5% fee
    }),
    { headers: { 'x-api-key': 'YOUR_API_KEY' } }
  )
).json();

// 2. Swap with fee account
const swapResponse = await (
  await fetch('https://api.jup.ag/swap/v1/swap', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'YOUR_API_KEY',
    },
    body: JSON.stringify({
      quoteResponse,
      userPublicKey: wallet.publicKey.toString(),
      feeAccount: 'YOUR_FEE_TOKEN_ACCOUNT', // Must be a token account for input or output mint
    }),
  })
).json();
```

The `feeAccount` must be an initialised token account owned by you. The token account mint must match either the input or output mint of the swap.

<Tip>
For detailed fee setup including creating the token account, see [Add Fees to Swap](/docs/swap/add-fees-to-swap).
</Tip>

## Complete example

A full, runnable script that swaps 1 SOL to USDC:

```ts
import {
  Connection,
  Keypair,
  VersionedTransaction,
} from '@solana/web3.js';
import fs from 'fs';

// Configuration
const API_KEY = 'YOUR_API_KEY';
const RPC_ENDPOINT = 'YOUR_RPC_ENDPOINT';
const API_BASE = 'https://api.jup.ag/swap/v1';

// Load wallet from file system
const wallet = Keypair.fromSecretKey(
  new Uint8Array(JSON.parse(fs.readFileSync('wallet.json', 'utf8')))
);
const connection = new Connection(RPC_ENDPOINT);

async function swap(
  inputMint: string,
  outputMint: string,
  amount: string,
  slippageBps: number
) {
  // Step 1: Get quote
  const quoteResponse = await (
    await fetch(
      `${API_BASE}/quote?` + new URLSearchParams({
        inputMint,
        outputMint,
        amount,
        slippageBps: slippageBps.toString(),
      }),
      { headers: { 'x-api-key': API_KEY } }
    )
  ).json();

  if (quoteResponse.error) {
    throw new Error(`Quote failed: ${quoteResponse.error}`);
  }

  console.log(`Route: ${quoteResponse.routePlan.map((r: any) => r.swapInfo.label).join(' → ')}`);
  console.log(`Output: ${quoteResponse.outAmount} (min: ${quoteResponse.otherAmountThreshold})`);

  // Step 2: Build transaction
  const swapResponse = await (
    await fetch(`${API_BASE}/swap`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': API_KEY,
      },
      body: JSON.stringify({
        quoteResponse,
        userPublicKey: wallet.publicKey.toString(),
        wrapAndUnwrapSol: true,
        prioritizationFeeLamports: {
          priorityLevelWithMaxLamports: {
            priorityLevel: 'veryHigh',
            maxLamports: 1000000,
          },
        },
      }),
    })
  ).json();

  if (swapResponse.error) {
    throw new Error(`Swap failed: ${swapResponse.error}`);
  }

  // Step 3: Sign and send
  const transaction = VersionedTransaction.deserialize(
    Buffer.from(swapResponse.swapTransaction, 'base64')
  );
  transaction.sign([wallet]);

  const signature = await connection.sendRawTransaction(transaction.serialize(), {
    maxRetries: 2,
    skipPreflight: true,
  });

  console.log(`Sent: https://solscan.io/tx/${signature}`);

  // Step 4: Confirm
  const confirmation = await connection.confirmTransaction(
    { signature },
    'finalized'
  );

  if (confirmation.value.err) {
    throw new Error(`Failed: ${JSON.stringify(confirmation.value.err)}`);
  }

  console.log('Swap confirmed.');
  return signature;
}

// Swap 1 SOL → USDC
swap(
```

## Common questions

<AccordionGroup>
  <Accordion title="Why is there no route found for this token?">
    Jupiter routes through 74+ DEXes on Solana. If no route is found, the token may not have liquidity on any supported DEX, or the token pair may not have a viable path. Check if the token is listed on a DEX and has sufficient liquidity.
  </Accordion>

  <Accordion title="Why is my transaction failing with slippage errors?">
    The price moved between when you got the quote and when the transaction landed. Re-quote with fresh pricing and send faster. For volatile tokens, consider increasing `slippageBps`.
  </Accordion>

  <Accordion title="Can I use Metis and Ultra together?">
    Yes. They access the same liquidity. You can use Ultra for simple swaps where you want zero-config execution, and Metis for flows that need custom instructions, CPI, or specific transaction control.
  </Accordion>

  <Accordion title="How do I filter which DEXes are used?">
    Use the `dexes` parameter to restrict routing to specific DEXes, or `excludeDexes` to block specific ones. Get the full list of DEX names from `GET https://api.jup.ag/swap/v1/program-id-to-label`.
  </Accordion>

  <Accordion title="What is the difference between /swap and /swap-instructions?">
    `/swap` returns a complete serialised transaction ready to sign and send. `/swap-instructions` returns individual instructions (compute budget, setup, swap, cleanup) that you can compose into your own transaction with custom logic. Use `/swap` for standard swaps; use `/swap-instructions` when you need to add custom instructions or call Jupiter via CPI.
  </Accordion>
</AccordionGroup>
