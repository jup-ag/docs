## HTTP status codes

| Code | Description           | Debug                                                                                       |
| :--- | :--------------------- | :------------------------------------------------------------------------------------------- |
| 200  | Success               | Request completed successfully.                                                             |
| 400  | Bad Request           | Check the request parameters and syntax.                                                    |
| 401  | Unauthorised          | Check that your API key is correct and active.                                              |
| 404  | Not Found             | Check the endpoint URL.                                                                     |
| 429  | Rate Limited          | You have exceeded your rate limit. Slow down requests, implement backoff, or [upgrade your plan](/portal/plans). |
| 500  | Internal Server Error | [Contact support](https://support.jup.ag) or reach out in [Discord](https://discord.gg/jup). |
| 502  | Bad Gateway           | [Contact support](https://support.jup.ag) or reach out in [Discord](https://discord.gg/jup). |
| 503  | Service Unavailable   | [Contact support](https://support.jup.ag) or reach out in [Discord](https://discord.gg/jup). |
| 504  | Gateway Timeout       | [Contact support](https://support.jup.ag) or reach out in [Discord](https://discord.gg/jup). |

## Rate limit headers

Every response includes headers to help you manage your request rate:

| Header | Description |
| :--- | :--- |
| `x-ratelimit-limit` | Maximum requests allowed in the current window |
| `x-ratelimit-remaining` | Requests remaining in the current window |
| `x-ratelimit-reset` | Unix timestamp when the current window resets |

See [Rate Limits](/portal/rate-limits) for details on limits per plan tier.
