The Swap API V2 routes through multiple liquidity sources. Which path you use (Meta-Aggregator or Metis Router) and which parameters you pass determines which routing engines are eligible, directly affecting the price you get.

## Routing engines

Jupiter's routing combines multiple engines and selects the best execution price across all of them, with a self-learning mechanism that automatically sidelines underperforming sources.

| Engine | Role |
| --- | --- |
| **Metis** | Jupiter's onchain routing engine. Multi-hop, multi-split swaps across Solana DEXes. Also available as an [independent public good](https://metis.builders). |
| **JupiterZ** | RFQ (Request for Quote) system where market makers compete to provide off-chain liquidity. Often beats onchain by 5-20bps on major pairs. Meta-Aggregator exclusive. |
| **Dflow** | Third-party decentralised order flow. |
| **OKX** | Third-party OKX liquidity. |

## Router availability by path

Which routers you get depends on which path you use:

| Path | Endpoint | Routers | Notes |
| --- | --- | --- | --- |
| **Meta-Aggregator** | `/swap/v2/order` (no optional params) | Metis, JupiterZ, Dflow, OKX | All engines compete. Best price. |
| **Meta-Aggregator** | `/swap/v2/order` (with optional params) | Depends on params | See [routing impact matrix](#routing-impact-matrix) below. |
| **Metis Router** | `/swap/v2/build` | Metis only | Onchain routing. No JupiterZ, no third-party routers. |
| **Swap V1** | `/v6/quote` + `/v6/swap` | Metis only | Legacy endpoint. Same Metis engine, no aggregation. |

## Why JupiterZ is Meta-Aggregator exclusive

JupiterZ (RFQ) is only available through the Meta-Aggregator path. Jupiter runs safety and validation mechanisms between integrators and market makers. Market makers have last-look execution rights, which requires controlled transaction handling to ensure validity and prevent spam. This means:

- Transactions routed through JupiterZ cannot be modified after `/order` returns them
- Use cases that require CPI, custom instructions, or any transaction modification must use the Metis Router path

<Note>
This is why choosing between Meta-Aggregator and Metis Router matters: the Meta-Aggregator gets you access to all liquidity sources (including RFQ market makers who often beat onchain by 5-20bps), but you give up transaction modification. The Metis Router gives you full control, but only through onchain routing.
</Note>

## Routing impact matrix

When using the Meta-Aggregator (`/order`), adding optional parameters can restrict which routers are eligible:

| Parameter | Metis | JupiterZ (RFQ) | Dflow | OKX |
| --- | --- | --- | --- | --- |
| **(no optional params)** | Yes | Yes | Yes | Yes |
| `receiver` | Yes | **No** | Yes | Yes |
| `referralAccount` & `referralFee` | Yes | **No** | Yes | Yes |
| `payer` (integrator gasless) | Yes | **No** | **No** | **No** |
| `excludeRouters: jupiterz` | Yes | **No** | Yes | Yes |

**Key takeaway:** adding `receiver`, `referralAccount`, `referralFee` or `payer` disables JupiterZ (RFQ), which may result in worse pricing on major pairs where market makers often beat onchain routing by 5-20bps.

This matrix only applies to the Meta-Aggregator. The Metis Router always uses Metis only, regardless of parameters.

For the full parameter reference, see the [API reference](/api-reference/swap).

## Order mode

The Meta-Aggregator `/order` response includes a `mode` field that tells you which routing behaviour was applied:

| Mode | Meaning | Routers |
| --- | --- | --- |
| **ultra** | No optional params used. All routers competed. Best price. | Metis, JupiterZ, Dflow, OKX |
| **manual** | Optional params detected. Routing may be restricted. | Depends on which params are set |

Use `mode` to verify your request got the routing behaviour you expected. If you see `"mode": "manual"` but didn't intend to restrict routing, check which optional parameters you are passing.

<Note>
This is similar to how the jup.ag frontend behaves: when you use custom settings like slippage, priority fee strategy, or dex/router exclusions, you get a different `mode`.
</Note>
