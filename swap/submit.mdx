`/submit` lets you send signed transactions through Jupiter's transaction landing infrastructure with SOL tips for priority processing. It runs on the same pipeline that powers Jupiter's own swap execution, giving you performant transaction landing for any transaction type.

Transaction submission can make or break your integration. If you are building with `/build` and managing your own transaction submission, consider running `/submit` alongside your existing submission path. If Jupiter's pipeline delivers even marginally better landing rates, that compounds into meaningful gains at scale.


## How /submit differs from /execute

Both endpoints submit transactions, but they serve different flows:

| | `/execute` | `/submit` |
|---|---|---|
| **Paired with** | `/order` (order-and-execute flow) | `/build` or any signed transaction |
| **Fee model** | Jupiter swap fees (included in the order) | SOL tips (minimum 0.001 SOL) |
| **Transaction source** | Must match the exact transaction from `/order` | Any valid signed Solana transaction |
| **Use case** | Recommended swap happy path | Custom transactions, `/build` transactions, non-Jupiter swaps |

## Quick start

The code example builds on the [`/build` endpoint](/swap/build). 

<Tip>
    The `/build` endpoint has a built-in param `tipAmount` to automatically add a transfer instruction to a randomised tip account.
</Tip>

<CodeGroup>
```typescript @solana/web3.js
const API_KEY = process.env.JUPITER_API_KEY!;
const BASE_URL = "https://api.jup.ag/swap/v2";

// Step 1: Call /build with tipAmount - tip instruction is auto-included
const buildRes = await fetch(
  `${BASE_URL}/build?` +
    new URLSearchParams({
      inputMint: "So11111111111111111111111111111111111111112",
      outputMint: "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",
      amount: "100000000",
      taker: signer.publicKey.toString(),
      tipAmount: "1000000", // 0.001 SOL minimum
    }),
  { headers: { "x-api-key": API_KEY } },
);
const build = await buildRes.json();

// Step 2: Assemble instructions - tipInstruction is already in the response
const instructions = [
  ComputeBudgetProgram.setComputeUnitLimit({ units: estimatedCUL }),
  ...build.computeBudgetInstructions.map(toInstruction),
  ...build.setupInstructions.map(toInstruction),
  toInstruction(build.swapInstruction),
  ...(build.cleanupInstruction
    ? [toInstruction(build.cleanupInstruction)]
    : []),
  ...build.otherInstructions.map(toInstruction),
  ...(build.tipInstruction
    ? [toInstruction(build.tipInstruction)]
    : []),
];

// Build and sign (see /build docs for full transaction assembly)
const message = new TransactionMessage({
  payerKey: signer.publicKey,
  recentBlockhash,
  instructions,
}).compileToV0Message(addressLookupTableAccounts);

const transaction = new VersionedTransaction(message);
transaction.sign([signer]);

// Step 3: Submit via /submit
const submitRes = await fetch("https://api.jup.ag/submit", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-api-key": API_KEY,
  },
  body: JSON.stringify({
    signedTransaction: Buffer.from(transaction.serialize()).toString("base64"),
  }),
});

const { signature } = await submitRes.json();
console.log(`Transaction submitted: https://solscan.io/tx/${signature}`);
```

## API reference

### POST /submit

```
POST https://api.jup.ag/submit
```

#### Request

| Field                | Type     | Description                              |
|---------------------|----------|------------------------------------------|
| `signedTransaction` | `string` | Base64-encoded signed transaction        |

#### Response (200)

```json
{
  "signature": "5UfDuX..."
}
```

#### Error response (400)

```json
{
  "error": "Transaction must contain at least 1000000 lamports tip to one of [...]"
}
```

#### Validation

- Transaction must be a valid signed Solana transaction
- All signatures must be valid
- Transaction must contain a SOL transfer of >= 1,000,000 lamports (0.001 SOL) to one of the 16 permitted tip receivers
- Transaction must not exceed max transaction size

## Standalone tip instructions

If you are submitting a non-Jupiter transaction (e.g. another swap protocol), add a tip instruction yourself. It is a standard SOL transfer to any one of the permitted tip receivers.

<Tip>
Randomise which authority you send to across transactions to reduce write-lock contention.
</Tip>

## Design decisions

| Decision          | Detail                                                              |
|-------------------|---------------------------------------------------------------------|
| Tip currency      | SOL only (for now)                                                  |
| Tip recipients    | 16 Jupiter V6 program authorities                                   |
| Minimum tip       | 1,000,000 lamports (0.001 SOL)                                      |
| Transaction types | Any signed Solana transaction                                       |
| API interface     | Follows `/execute` pattern, not Solana RPC `sendTransaction`        |
